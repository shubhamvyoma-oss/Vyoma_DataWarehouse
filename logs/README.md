# logs/

Runtime log files generated by the API scripts. This directory is gitignored — logs are never committed to the repository.

---

## Files

| File | Generated by | Contents |
|---|---|---|
| `attendance_backfill.log` | `fetch_attendance.py` (stdout) | Progress output: dates processed, Bronze rows inserted, Silver rows upserted, rate limit sleeps, final summary |
| `attendance_backfill.err` | `fetch_attendance.py` (stderr) | Any uncaught exceptions or error output |
| `attendance_backfill.log.err` | Legacy stderr redirect | Kept for reference; typically empty |

The webhook receiver writes its own log to `ingestion/webhook_receiver.log` (PII-masked, rotated).

---

## Log Format (attendance_backfill.log)

```
==================================================
ATTENDANCE PIPELINE  (report_type=55)
==================================================
Dates to pull: 846
  2024-01-06  ..  2026-04-30

  [2024-01-15] API rows=6471 | Bronze +6470 | Silver +4 | staff=1
  [2024-01-25] API rows=3441 | Bronze +3440 | Silver +3 | staff=1
  Rate limit: sleeping 60s after 25 calls ...
  ...
  ATTENDANCE PULL COMPLETE
  --------------------------------------------
  Dates pulled      : 846
  Dates failed      : 0
  Bronze rows added : 1,234,567
  Silver rows added : 8,920
  Batches covered   : 312
  Bundles covered   : 198
  Date range        : 2024-01-06 -> 2026-04-30
  Staff rows skipped: 1,234 (@vyoma emails)
  --------------------------------------------
```

Every 10th date is printed. Rate limit sleeps are always printed. The final summary always prints.

---

## How to Start the Backfill with Logging

```bash
python -u api_scripts/fetch_attendance.py --start 2024-01-01 --end 2026-04-30 > logs/attendance_backfill.log 2> logs/attendance_backfill.err
```

The `-u` flag disables Python's stdout buffering so log lines appear in real time.

---

## Checking Backfill Progress

```bash
tail -20 logs/attendance_backfill.log
```
